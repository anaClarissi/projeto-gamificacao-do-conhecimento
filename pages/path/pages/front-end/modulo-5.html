Front End

Módulo 5 — Design e Boas Práticas

Este módulo ensina como criar interfaces bonitas, organizadas e profissionais usando princípios de design e boas práticas de front-end. É aqui que o aluno aprende como deixar o site agradável, limpo, legível e intuitivo, mesmo sem ser designer.

1. O que é Design no Front-End?

Design no front-end não é só deixar o site “bonito”. É tornar tudo funcional, claro, equilibrado e fácil de entender.

Um site bem projetado:

  • É fácil de navegar
  • Usa cores que combinam
  • Tem textos legíveis
  • Tem espaçamentos que deixam tudo organizado
  • Funciona bem em telas grandes e pequenas

Você não precisa ser designer — apenas seguir princípios simples.

2. Tipografia — Como escolher e usar fontes

A tipografia define a forma como o texto aparece no site.

  • Tamanho e hierarquia

    Use tamanhos diferentes para indicar o que é mais importante:

    • Título principal (h1): maior
    • Subtítulos (h2, h3): médios
    • Texto comum (p): tamanho confortável

    Regra: títulos grandes → chamam atenção; textos pequenos demais cansam a leitura.

  • Peso da fonte
    • bold: destaque
    • regular: leitura normal
  • Espaçamento entre linhas

    Sites profissionais sempre usam espaços maiores entre linhas para que o texto “respire”.

    Exemplo CSS:

    css

    p {
    line-heigth: 1.6;
    }

Cores — Como escolher paletas harmônicas

Paleta principal

Use 1 cor principal e 1 ou 2 cores secundárias.

Ex.:

  • Cor primária: azul
  • Secundária: verde
  • Neutra: cinza

Cores de destaque

Use cores fortes apenas para:

  • Botões importantes
  • Avisos
  • Chamada para ação (CTA)

Acessibilidade

Certifique-se de que o texto contraste com o fundo para facilitar a leitura.

Exemplo:

Texto branco em fundo amarelo → ruim

Texto branco em fundo azul escuro → ótimo

Ferramenta útil (não citei links): verificador de contraste de cores (o aluno pode pesquisar no Google).

Layout — Organização visual da página

Espaçamentos

Espaçamentos deixam o conteúdo organizado.

Exemplo:

css

section {
padding: 20px;
}

Alinhamento

Alinhar corretamente torna o site mais limpo:

  • Textos devem estar alinhados à esquerda
  • Botões alinhados
  • Imagens centralizadas (quando necessário)

Composição do layout

Use estruturas que guiam o olhar do usuário:

  • Colunas
  • Cards
  • Containers
  • Divisões claras

5. Componentes Visuais

São elementos prontos que melhoram o design:

  • Cards
  • Botões estilizados
  • Barras de navegação
  • Caixas com sombra
  • Inputs com bordas suaves

Exemplo de card:

html

<div class="card">
<h3>Título</h3>
<p>Descrição do card.</p>
</div>

css

.card {
padding: 15px;
border-radius: 10px;
box-shadow: 0 2px 6px rgba(0, 0, 0, 0.15);
}

6. Boas Práticas de HTML

  • Feche todas as tags
  • Evite misturar estilos dentro do HTML
  • Nomeie classes claramente
  • Use indentação
  • Use semântica corretamente (h1, p, nav, header, section)

Exemplo ruim:

html

<div> <p> ola </p> </div>

Exemplo bom:

html

<section>
<p> Olá </p>
</section>

Boas Práticas de CSS

Organização

Organize o CSS por blocos:

  • Reset
  • Layout
  • Componentes
  • Botões
  • Responsividade

Evite repetir código

Se dois elementos têm o mesmo estilo → crie uma classe.

Responsividade

Use medidas flexíveis:

  • %
  • em
  • rem
  • vh
  • vw

8. Responsividade e Mobile First

Sites devem funcionar bem em celulares, tablets e computadores.

O que é mobile first?

Você começa o layout pelo celular, depois aumenta para telas maiores.

Media Queries

css

@media (min-width: 768px) {
/* estilos para telas maiores */
}

9. Experiência do Usuário (UX)

UX é como o usuário se sente usando o site.

  • É fácil encontrar o que procura?
  • A navegação é clara?
  • Os botões parecem clicáveis?
  • Tem feedback visual (ex.: botão muda de cor ao clicar)?

Essas pequenas coisas fazem o site parecer profissional.

10. Checklist Final do Design Profissional

Antes de finalizar seu site, revise:

  • Tipografia consistente
  • Paleta de cores bem definida
  • Layout organizado
  • Botões destacando ações importantes
  • Componentes reutilizáveis
  • Responsividade funcionando
  • Cores com bom contraste
  • Espaçamentos adequados
  • Uso correto de HTML semântico
